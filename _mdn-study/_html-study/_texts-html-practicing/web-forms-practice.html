Abstract and useful informations

  • HTML form text fields are simple plain text input controls. This means that you cannot use them to perform rich text editing (bold, italic, etc.). All rich text editors you'll encounter are custom widgets created with HTML, CSS, and JavaScript;
  • Single line text fields have only one true constraint: if you type text with line breaks, the browser removes those line breaks before sending the data to the server;
  • After send a data in a password type input it will be sent in a plain text, being horrible in secure issues;
  • <input> only accepts plain text as content;
  • If the image button is used to submit the form, this control doesn't submit its value â€” instead, the X and Y coordinates of the click on the image are submitted (the coordinates are relative to the image, meaning that the upper-left corner of the image represents the coordinate (0, 0)). The coordinates are sent as two key/value pairs:
    • The X value key is the value of the name attribute followed by the string ".x".
    • The Y value key is the value of the name attribute followed by the string ".y".
  • Use the boolean attribute "multiple" to allow the user to select multiple files with a file type input.
  • On some mobile devices, the file picker can access photos, videos, and audio captured directly by the device's camera and microphone by adding capture information to the accept attribute, using some of these values "camera", "camcorder" and "microphone".
  • Only one form-associated element can have an autofocus boolean attribute.
