CSS Specificity

What is Specificity?

Se há uma ou mais regras de CSS conflitantes que se dirigem a um mesmo elemento, o navegador seguirá algumas regras para determinar qual é mais específica e, portanto, a que será considerada.

Pense na especificidade como uma pontuação/classificação que determina qual declaração de estilo será aplicada, em última instância, a um elemento.

O seletor universal (*) tem uma pequena especificidade, enquanto o seletor por ID é altamente específico.

A especificidade costuma ser a razão de uma regra CSS não ser aplicada para algum elemento, embora pareça que deveria ser.

Specificity Hierarchy

Cada seletor tem seu lugar na hierarquia de especificidade. Há 4 categorias que definem o nível de especificidade de um seletor:

Inline styles - Um estilo inline é anexado diretamente ao elemento a ser estilizado, dentro da tag usando "style=''".

Ids - Um ID é um indetificador único para os elementos da página, como por exemplo: #navbar.

Classes, attributes and pseudo-classes - Esta categoria inclui: .classes, [attributes] e pseudo-classes como :hover, :focus, etc.

Elements and pseudo-elements - Esta categoria inclui nome de elementos e pseudo-elementos como por exemplo: h1, div, :before and :after.

How to Calculate Specificity?

É possível calcular, literalmente, o nível de especificidade de um elemento, veja como fazer isso a seguir.

Comece com 0, adicione 1000 para o atributo de estilo, adicione 100 para cada ID e adicione 10 para cada atributo, classe ou pseudo-classe e adicione 1 para cada nome de elemento ou pseudo-elemento.

Considere os 3 fragmentos de códigos a seguir:

A: h1

B: #content h1

C: <div id="content"><h1 style="color: #ffffff">Cabeçalho</h1></div>

A especificidade de A é 1 (um elemento)

A especificidade de B é 101 (uma referência ID e um elemento)

A especificidade de C é 1000 (inline styling)

Como 1 < 101 < 1000, a terceira regra (C) tem um nível maior de especificidade e, portanto, será aplicado.

Specificity Rules

Especificidade igual: a última regra conta - Se a mesma regra for duas vezes numa folha de estilo externa, a última (a que está mais abaix) será aplicada.

Seletores de ID tem uma especificidade maior do que um seletor de atributo - div#a > #a > div[id=a].

Seletores contextuais são mais específicos do que um único seletor de elemento - As estilizações feitas dentro da tag style no arquivo HTML estão mais próximas dos elementos, portanto, elas vencem as estilizações feitas externamente.

Um seletor de classe vence qualquer número de seletores de elemento - Um seletor de classe, como .container vence div, h1, p, etc.

O seletor universal e valores herdados tem especificidade de 0 - *, body * e similares tem um nível de especificidade de 0. Valores herdados também tem uma especificidade de 0.
